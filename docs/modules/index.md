## How Each Module Works

Every module follows the same **3-part structure**:

1. **📖 Theory** — The distributed systems concept from DDIA, explained clearly
2. **🧠 Mental Model** — How Temporal addresses this problem and why
3. **🛠️ Build It** — TypeScript code you write, run, break, and fix

---

## Recommended Path

Modules are designed to be completed **in order**. Each module builds on concepts and code from previous modules.

If you're already familiar with distributed systems theory, you can skim the theory sections and focus on the Temporal implementation.
